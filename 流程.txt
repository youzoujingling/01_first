

一、创建本地管理
1.进入文件夹
2.进行初始化
 git init
3.检测文件夹状态,文件颜色为红色
 git status 
4.将文件提交到暂存区
 git add 1.txt --某个文件
 git add .    --所有文件
5.再次检查文件状态，文件颜色为绿色
 git status 
6.从暂存区提交到版本库
 git conmmit -m 'v1' [文件名]
7.设置提交身份验证
  git config --global user.email "3335029085@qq.com"
  git config --global user.name "syx"
8.再次提交
9.再次检查文件状态，无文件
 git status 
10.检查提交记录
 git log [文件名]



二、版本切换
1.从版本库某一个版本号切换到版本库另一个版本号（HEAD指向该版本），此时工作区也切换
 git reset --hard 版本号
 单独回滚一个文件没有（需要记录）


2.查看版本提交记录
  git reflog 文件名     --可查看所有提交记录
  git log    文件名     --只能查看HEAD之前的记录


三、回滚(必须是已经提交过的，做了修改才能回滚；只是建立了文件，然后修改了，没有加入到版本中会报找不到该文件)
1.从版本库->暂存区
  git reset --soft 版本号
2.从暂存区->工作区（变动）
  git reset HEAD 1.txt
3.从工作区（变动）->工作区（静态）
 git checkout -- 1.txt
4.从版本库->工作区（变动）
 git reset --mix 版本号
5.从版本库->工作区（静态）
 git reset --hard 版本号
